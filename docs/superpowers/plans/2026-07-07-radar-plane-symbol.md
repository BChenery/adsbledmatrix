# Radar Plane Symbol Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a `use_plane_symbol` option to the existing `radar` layout element so the aircraft marker can be rendered as a small rotating plane silhouette instead of a dot.

**Architecture:** Extend the existing radar renderer in both the backend (`DisplayEngine._draw_radar`) and the Designer canvas (`Canvas.tsx`) with the same simple plane polygon. Add a new boolean property to the frontend type and expose it as a checkbox in the property panel. A backend unit test verifies the rotated plane is drawn at the expected location.

**Tech Stack:** Python/PIL (backend), React/TypeScript/Tailwind (frontend), FastAPI for serving the built frontend.

  ```python
          angle = math.radians(angle_deg)

          sin_a = math.sin(angle)
  ```

  ```typescript
  function rotatePoint(px: number, py: number, cx: number, cy: number, angleDeg: number): [number, number] {
    const angle = (angleDeg * Math.PI) / 180;
    const cos = Math.cos(angle);
    const sin = Math.sin(angle);
    return [
      cx + (px - cx) * cos - (py - cy) * sin,
      cy + (px - cx) * sin + (py - cy) * cos,
    ];
  }
  ```
